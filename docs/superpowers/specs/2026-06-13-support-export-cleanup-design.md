# Support Export Cleanup Design

## Context

The desktop shell now exports support bundles, shows the saved path, opens the support directory, and restores the last export record on launch. The support directory can still grow forever because each export writes a timestamped `keli-support-*.json` file and keeps `last-support-export.json` as metadata.

The user has also flagged disk pressure before, so diagnostics should expose a safe cleanup path inside the client.

## Goal

Show support export directory usage in the diagnostics UI and provide a safe one-click cleanup action for files generated by Keli support export.

## Design Brief

- Product: existing Keli native desktop shell.
- Visual source: current Chinese diagnostics support panel in `crates/keli-desktop-shell/src/html.rs`.
- Interaction level: fully functional desktop UI, IPC event, filesystem cleanup, status updates, and tests.

## Options

Recommended: clean only Keli-generated support export artifacts in the default support directory.

Tradeoff: it is conservative and safe. It deletes timestamped support bundles and `last-support-export.json`, but it does not recurse or touch unrelated files that happen to live in the same directory.

Alternative: delete the whole support directory.

Tradeoff: simpler code, but riskier. If a user manually placed notes or copied files there, the client would remove them.

Alternative: keep the newest support bundle and delete older ones.

Tradeoff: friendlier for support workflows but more complex, because the UI must explain retention and the code must sort timestamps or metadata.

## Scope

In scope:

- Count support export artifacts in the default support directory.
- Sum their byte size.
- Show the count and total size in both support export panels.
- Add a Chinese "清理旧支持包" button.
- Delete only:
  - files whose filename starts with `keli-support-`
  - `last-support-export.json`
- After cleanup, reset the last export display to "尚未生成" and disable the open-directory action.

Out of scope:

- Recursive directory cleanup.
- Cleaning logs, caches, build artifacts, or installer outputs.
- User-selectable retention policies.
- Confirm dialogs; this desktop shell does not yet have modal infrastructure, and the safe filename allowlist keeps blast radius narrow.

## Architecture

`crates/keli-desktop-shell/src/support.rs` owns artifact discovery and deletion:

- `support_export_artifacts(directory)` returns allowed support export files.
- `summarize_support_export_directory(directory)` returns a serializable summary with `status`, `directory`, `file_count`, and `byte_count`.
- `clear_support_export_directory(directory)` deletes allowed files and returns `SupportExportCleanupSummary` with deleted count, reclaimed bytes, remaining count, and directory.

`crates/keli-desktop-shell/src/actions.rs` adds an IPC event:

- Message: `clear-support-exports`
- Event: `DesktopShellUiEvent::ClearSupportExports`

`crates/keli-desktop-shell/src/html.rs` adds UI status and script helpers:

- `window.keliSetSupportStorage(summary)` updates directory usage lines in both support panels.
- `window.keliSetSupportCleanup(summary)` updates cleanup result, resets last export state when files were removed, and mirrors the global operation status.
- The diagnostics support panel gets a "清理旧支持包" button next to export/open/copy.

`crates/keli-desktop-shell/src/main.rs` wires the event:

- Startup sync reads the current directory usage after restoring the last record.
- Export sync updates both last export status and directory usage.
- Cleanup sync deletes support artifacts, updates storage/cleanup scripts, and refreshes the shell snapshot.

## Error Handling

- Missing support directory summarizes as zero files and zero bytes.
- Per-file metadata errors are ignored for summary because cleanup should still work with the rest of the directory.
- Cleanup returns an error if deleting an allowed file fails; the UI shows an error status.
- Cleanup does not remove unrelated files.

## Testing

Add support module tests for:

- Summary counts only Keli support artifacts and ignores unrelated files.
- Cleanup deletes Keli support artifacts and preserves unrelated files.
- Missing directory summary returns zero.

Add action and HTML tests for:

- IPC maps `clear-support-exports`.
- Diagnostics support panel includes storage status and cleanup button.
- Storage and cleanup status scripts serialize the expected payloads.

Add main smoke coverage:

- Desktop smoke reports the cleanup entrypoint.
- Support export smoke still passes after persistence and cleanup code are present.
